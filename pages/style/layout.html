Layout

Display

The display property defines how an element behaves in the layout (whether it is a block, inline, flex container, etc.). When you combine position: absolute; with different display values, it influences how the element behaves within the layout and interacts with its parent or container.

  • .display-flex - The .item elements inside it will be arranged horizontally (since the default is flex-direction: row) and spaced out evenly.
  • .display-none - The display: none; value removes an element completely from the document flow.
  • .display-block - Makes an element block-level, taking up the full width of its container and starting on a new line.
  • .display-grid - The grid value turns an element into a grid container, which allows you to create two, three or four dimensional layouts.

Use Cases of Display

  • Be cautious when using display: none, as it can make elements inaccessible to screen readers. Consider using visibility or opacity if accessibility is a concern.
  • For complex layouts, Flexbox and Grid are preferred over older methods like float or inline-block.
  • Use display: flex or display: grid to center modals or overlays on the screen.
  • The display: flex property is used to create flexible layouts. It allows child elements to be aligned and distributed dynamically within a container.
  • Elements with display: block occupy the entire width of their container, creating line breaks before and after the element.

Postion

The position property is used to control the positioning of an element within a page or container. It determines how the element is placed on the screen and how it interacts with other elements. The position property works in conjunction with properties like top, right, bottom, and left to define the element's exact location.

  • .position-absolute - Position absolute property is used to position an element relative to its nearest positioned ancestor (an ancestor element that has any position value other than static).
  • .position-fixed - Fixed positioning removes the element from the normal document flow and positions it relative to the viewport (the browser window). The element will stay in the same position on the screen even when the page is scrolled.
  • .position-static - The position: static property is the default positioning behavior for most elements. When an element has position: static, it follows the normal flow of the document, meaning it is positioned based on where it appears in the engage it's structure, and its position cannot be modified using the top, right, bottom, or left properties.
  • .position-relative - When you set an element to position: relative;, it is positioned relative to its original position in the document flow. It still takes up space in the layout, but you can adjust its position using the top, right, bottom, and left properties.
  • .position-sticky - Sticky positioning is a hybrid between relative and fixed positioning. The element is treated as relative until it reaches a specified position (usually using top, right, bottom, or left), at which point it becomes fixed within the viewport. The element sticks to its position as the page scrolls until it reaches its containing block (e.g., a parent element or a defined scrollable area).

Use Cases of Position

  • relative: Allows an element to be positioned relative to its normal position.
  • absolute: Positions an element relative to the nearest positioned (non-static) ancestor.
  • fixed: Positions an element relative to the viewport and remains fixed during scrolling.
  • Use position: sticky for navigation bars or headers to maintain their visibility as users scroll.
  • Sticky table headers also enhance usability by keeping column titles accessible while viewing large datasets.
